for (var p in button) button[p].className = ‘’;

className 属性

className 属性设置或返回元素的 class 属性。

classList 和 className 的联系和区别：

className 设置 class 属性时，是“一刀切”的。比如：想删除元素的某个类名，需要获取全部类名，在全部类名中找到想删除的，删除后，再将整个类名字符串赋给元素。

HTML 5 的 classList 就会方便很多，其 API 有：

  • length：表示类名的个数

  • item(index)：返回 index 索引对应的类名，如果 index 超出范围，则返回 null

  • add(class)：新增一个类名，如果类名已存在，则添加忽略

  • remove(class)：删除一个类名

  • toggle(class)：若类名已存在，则删除，返回 false；若不存在，则添加该类名，返回 true

  • contains()：检查是否存在此类名，弱存在，则返回 true，若不存在，则返回 false

  • toSting()：等同于 className

然而，classList 属性的浏览器兼容性并不乐观，希望不久的将来能够完全支持。在 PC 上，可以通过 document.body.classList === undefined 来判断浏览器是否支持。

for…in

for...in 语句用于遍历数组或者对象的属性（对数组或者对象的属性进行循环操作）。

getElementsByTagName() 返回的是一个 NodeList，并不是 Array。MDN 中建议：

不要尝试使用 for...in 或者 for each...in 来遍历一个 NodeList 对象中的对象，因为如果你把 length 和 item 也看成 element 对象的话，NodeList 对象中的 length 和 item 属性也会被遍历出来，这可能会导致你的脚本运行出错。此外，for...in 不能保证访问这些属性的顺序。
